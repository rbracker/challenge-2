My Work

The Quiz-time project involves building a browser-based, timed

coding quiz focusing on JavaScript fundamentals. The goal is to provide a practical experience

in handling coding assessments, a common part of web development interviews. The app will feature

dynamically updated HTML and CSS powered by JavaScript, creating a polished and responsive user

interface. Users, particularly coding boot camp students, can click a start button to initiate the timed

quiz. As they answer questions, the timer starts, and incorrect answers result in time deductions. The game

concludes when all questions are answered or the timer reaches zero. After the game ends, users can save their

initials and score, allowing them to gauge their progress and compare results with peers. The project aims to

prepare students for technical interviews by providing a hands-on opportunity to apply JavaScript skills and

simulate a real-world coding assessment scenario.

The Elemental-Forecast project involves creating a dynamic weather dashboard

that runs in the browser, utilizing third-party APIs, specifically the OpenWeatherMap API. The dashboard

allows travelers to access weather outlooks for multiple cities, aiding in trip planning. Users can search for a

city, and upon searching, the dashboard displays current and future weather conditions for the selected city, while

also adding it to the search history. Details provided include the city name, date, weather icon, temperature,

humidity, and wind speed. Additionally, the dashboard presents a 5-day forecast with date, weather icon, temperature,

wind speed, and humidity. The search history allows users to revisit the weather conditions for a previously searched

city. The project employs localStorage to store persistent data and requires the use of a valid API key for access to

the OpenWeatherMap API.

The Dad-A-Base serves as a crucial milestone in my full-stack web development journey.

Key requirements include the use of a CSS framework other than Bootstrap, deployment on GitHub Pages, interactivity,

incorporation of at least two server-side APIs, avoidance of alerts/confirms/prompts (favoring modals), implementation of client-side

storage for persistent data, responsiveness, and the delivery of a polished UI.

The project is more than just a demonstration of technical skills; it showcases my ability to collaborate effectively with other

developers.
